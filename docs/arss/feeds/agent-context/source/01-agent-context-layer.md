# The agent context layer

Agents do not browse the web like humans. They ingest context, retrieve it later, cite it when useful, and forget it when policy says it has expired. That means the web needs a context layer: feeds that are machine-readable, publisher-controlled, rights-aware and payable.

ARSS is the feed format for this layer. It keeps the RSS subscription shape but adds agent-facing terms: cache TTL, allowed uses, denied uses, attribution requirements, provenance hashes and x402 payment metadata.

The product wedge is simple: subscribe your agent to sources you trust. The protocol ambition is larger: make the web legible, payable and accountable to agents.
